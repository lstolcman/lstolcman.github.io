podman and docker-compose in Ubuntu 22.04.3 (WSL2)

11 Jan 2024

Ubuntu reports fairly old version od podman in its default apt registry.

Here I will present how to install podman 4.6.2 from separate apt. Afterwards, we’ll also configure podman socket, install docker-compose and run example scripts.

System

I am using wsl2 with ubuntu 22 lts:

➜  lsb_release -a
No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 22.04.3 LTS
Release:        22.04
Codename:       jammy

Adding apt repository

Stock ubuntu does have podman but in ancient version 3.x something. To install newer version using apt, we need to add a kubic repository, as suggested in podman readme.

The source is here.

Execute following lines in a console:

echo 'deb http://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/unstable/xUbuntu_22.04/ /' | sudo tee /etc/apt/sources.list.d/devel:kubic:libcontainers:unstable.list
curl -fsSL https://download.opensuse.org/repositories/devel:kubic:libcontainers:unstable/xUbuntu_22.04/Release.key | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/devel_kubic_libcontainers_unstable.gpg > /dev/null
sudo apt -y update
sudo apt -y install podman

Setting up podman socket service

The detailed instruction is here

systemctl --user status podman.socket podman.service
systemctl --user enable podman.socket
systemctl --user start podman.socket
systemctl --user status podman.socket podman.service

downloading docker-compose

docker-compose is a singlefile binary. The following script downloads it into local bin and makes it executable:

wget https://github.com/docker/compose/releases/download/v2.23.3/docker-compose-linux-x86_64 -O /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

example run of compose

export DOCKER_HOST=unix://$XDG_RUNTIME_DIR/podman/podman.sock

cat > compose.yml<< EOF
services:
  base:
    image: alpine:latest
    command: sh -c "echo ok"
EOF

cat compose.yml

docker-compose up
